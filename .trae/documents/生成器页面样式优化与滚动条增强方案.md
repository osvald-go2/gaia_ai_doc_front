## 问题定位
- 左/中列使用 `@radix-ui/react-scroll-area` 但未渲染滚动条组件，且根节点设置了 `overflow-hidden`，导致页面没有可见滚动条：`src/pages/GeneratorLayout.tsx:85-96`、`src/pages/GeneratorLayout.tsx:110-126`。
- 分栏分割线使用 `PanelResizeHandle`，当前宽度为 `w-1`（约 4px），视觉偏粗：`src/pages/GeneratorLayout.tsx:99`、`src/pages/GeneratorLayout.tsx:129`。
- 右侧编辑区内部已有自定义滚动条（`InterfaceEditor.tsx:432-437`），需避免产生“双滚动”。

## 优化目标
- 为生成器页面三列内容提供一致、精致的滚动条展示。
- 将分栏分割线优化为更细、更现代的样式，保留良好的可拖拽反馈。
- 保持现有主题变量与组件风格一致，不引入突兀的颜色或厚重边框。

## 调整方案
- 滚动条统一：左/中列改用项目内封装的 `ScrollArea` 组件（`src/components/ui/scroll-area.tsx`），其已内置 `Viewport` 与 `ScrollBar`，无需手动拼装；去除根节点的 `overflow-hidden`。
- 分割线统一：将 `PanelResizeHandle` 替换为 `ResizableHandle`（`src/components/ui/resizable.tsx`），该组件默认 `w-px` 细线并可选 `withHandle` 显示握柄，hover 反馈更精致。
- 右侧编辑区：外层不再额外包滚动，保留 `InterfaceEditor` 内部滚动逻辑，避免“双滚动”。

## 具体修改
- 文件：`src/pages/GeneratorLayout.tsx`
  - 替换导入：移除 `* as ScrollAreaPrimitive`；新增 `import { ScrollArea } from "../components/ui/scroll-area"` 与 `import { ResizableHandle } from "../components/ui/resizable"`。
  - 左列：`ScrollAreaPrimitive.Root` → `ScrollArea`，类名改为 `className="flex-1 min-h-0"`，去掉 `overflow-hidden`；内部去掉 `Viewport` 包装，直接渲染 `DocumentPanel`。
  - 中列：同左列处理，`InterfaceList` 由 `ScrollArea` 包裹，显示滚动条。
  - 分割线：`<PanelResizeHandle className="w-1 ..." />` → `<ResizableHandle withHandle />`，无需手动设定厚度，使用组件默认更细的样式与 hover 效果。
  - 其余布局尺寸与 `min-h-0` 保持不变，避免破坏现有自适应。
